## 列4 · 立项方案（附链接 + 四段）

> 📎 链接：完整版立项书见本仓库 `立项表.docx` / `立项表.md`，把仓库 URL 填到此列。
> （完整版含架构图、跨域能力地图、结论引擎、杀手锏案例、里程碑、风险应对）

**① 痛点**
经营分析"盲人摸象"：获取看渠道、行为看漏斗、供给看库存、物流看时效，**没人能同时看全，增长卡点掉在团队的缝里**；外部热点/竞品/大促实时变，"内部多域+外部动态"超出人脑工作记忆。结果：决策慢、归因扯皮、拉新预算（CAC）白烧。

**② 方案**
锚点指标 + 三层架构：语义层(指标字典) → 工具层(每个域=一张宽表+一个查询，原始数据留本地、只聚合结果进 LLM) → 跨域归因引擎(分品类基准·样本量门槛·转化缺口 z 检验·混杂因子守门·RICE 优先级·单位经济) → 封装为 Claude Code Skill，按 **7 类方法论结论体系**(AARRR/归因/单位经济/RICE/增长循环/盲区/验证设计)产出诊断与打法。**新增一个域=加一张宽表+一个工具，域可无限扩展。**

**③ 价值**
- 归因从跨团队 0.5–2 天 → **分钟级**；
- 可量化"可挽回 GMV / 止损白烧 CAC"（MVP 已 ~$35万 / ~$1.8万）；
- **扁平化协作**：一个 agent 顶过去多团队横向拉通会；
- **资产沉淀**：增长方法论固化为可复用、可审计的 context，反哺大模型；
- 可平移复购、大促冷启动、品类增长、商家经营参谋 skill。

**④ 预期成果**
可演示 Skill `/growth-diagnose`（现场出诊断）· 真实数据前后对比（耗时/可挽回单/转化提升）· 供需错配象限图可视化 · Git 提交记录（真实开发过程）· 演示视频。
