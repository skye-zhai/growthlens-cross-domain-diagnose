# AGENTS.md —— 给 AI 评审 / agent 的核对指引

本文件供**自动化评审（AI / agent）**快速、确定性地核验本仓库。人类评委请看
[README.md](README.md) 与 [评委导览.md](评委导览.md)；本文件不改变任何对外叙述，
只提供"如何一键复算、断言、对照"的操作入口。

---

## 1. 一条命令：复现 + 自检

```bash
bash run.sh
```

它会：建虚拟环境 → 装依赖（仅 pandas、pyyaml）→ 固定种子造数 → 跑分析 →
跑 `verify.py`（把每个对外数字复算并断言）。**任一步失败即非零退出。**
最后一节打印 `全部 20 条声明均由引擎确定性复算通过 ✓` 即为通过。

等价入口：`make`（= `make all`）。只想跑断言：`make verify`。

预期：进程退出码 `0`，且输出含 `全部 20 条声明均由引擎确定性复算通过 ✓`。

## 2. 不变量（评审可据此判定真伪）

| 不变量 | 期望 | 在哪验证 |
|---|---|---|
| 确定性 | 固定种子 `SEED=42`（`gen_synthetic.py`），两次造数字节级一致 | `verify.py` 末条（md5 比对） |
| 数字非编造 | 所有对外 headline 数字 = 引擎 `analyze.run()` 实算值 | `verify.py` 全表 |
| 明细不进 LLM | `tools.py` 只返回聚合结果；原始 event 留本地 | 读 `tools.py` / `semantic.yaml` |
| 不误报（红鲱鱼） | 巴西×美妆"转化低但供给健康"→ 供给归因强度被压到 ≈0.36，主因判为"非供给" | `verify.py` ⑤ |
| 区分力 | 存在"统计显著但被混杂降权"的格子（纯漏斗做不到）≥1（实算 12） | `verify.py` ⑥ |

## 4. 数据说明（合规与边界）

- `data/*.csv` 为**合成数据**（`gen_synthetic.py`，固定种子），用于演示，内埋
  一个真洞察（泰国×3C 供给卡点）+ 一个红鲱鱼（巴西×美妆 非供给）。
- 真实使用：把同 schema 的预聚合宽表放进 `data/`，无需改代码（见 README「换成真实数据」）。
- 设计上**原始明细不进 LLM**，仅聚合结果进入，天然合规。

## 5. 文件地图（核对从哪读起）

| 文件 | 作用 |
|---|---|
| `run.sh` / `Makefile` | 一键复现 + 自检入口 |
| `verify.py` | 声明自检：复算每个对外数字并断言（核对真伪从这里） |
| `semantic.yaml` | 语义层：指标口径、维度、跨域归因方法 |
| `tools.py` | 工具层：pandas 查询 + 跨域 join（统计显著性 / 混杂守门 / RICE / 单位经济） |
| `analyze.py` | 诊断主流程，`run()` 返回结构化结果，`--json` 输出 |
| `gen_synthetic.py` | 合成数据生成器（SEED=42，内埋真洞察 + 红鲱鱼） |
| `chart.py` | 象限图（可选，需 `requirements-viz.txt` 里的 matplotlib） |
| `build_poster.py` | 海报（可选）：生成 `submission/海报.html`，并在有 Chrome 时渲染 `output/poster.png` |
